一、安装要求

1、机器要求

在开始之前，部署Kubernetes集群机器需要满足以下几个条件：

  • 一台或多台机器，操作系统 CentOS7.x-86_x64
  • 硬件配置：2GB或更多RAM，2个CPU或更多CPU，硬盘30GB或更多
  • 集群中所有机器之间网络互通
  • 可以访问外网，需要拉取镜像，如果服务器不能上网，需要提前下载镜像并导入节点
  • 禁止swap分区

确保所有节点系统时间一致

# yum install ntpdate -y
# ntpdate time.windows.com

二、下载所需文件并修改配置

1、下载Ansible部署文件

# git clone https://github.com/calvinGithub/ansible-install-k8s
# cd ansible-install-k8s

下载软件包并上传到上面列举的机器root目录下，并解压压缩包

链接:https://pan.baidu.com/s/11-c6ZEwwKS2YsnZqlcMIyw 密码:gsep

# tar zxf binary_pkg.tar.gz

2、修改Ansible文件

修改hosts文件，根据规划修改对应IP和名称。

# vi hosts

3、修改group_vars/all.yml文件

修改软件包目录和证书可信任IP。

# vim group_vars/all.yml

# 安装目录 
software_dir: '/root/binary_pkg'
k8s_work_dir: '/opt/kubernetes'
etcd_work_dir: '/opt/etcd'
tmp_dir: '/tmp/k8s'

# 集群网络
service_cidr: '10.0.0.0/24'
cluster_dns: '10.0.0.2'   # 与roles/addons/files/coredns.yaml中IP一致，并且是service_cidr中的IP
pod_cidr: '10.244.0.0/16' # 与roles/addons/files/kube-flannel.yaml中网段一致
service_nodeport_range: '30000-32767'
cluster_domain: 'cluster.local'

# 高可用，如果部署单Master，该项忽略
vip: '10.1.34.74'
nic: 'ens192' 

# 自签证书可信任IP列表，为方便扩展，可添加多个预留IP
cert_hosts:
  # 包含所有LB、VIP、Master IP和service_cidr的第一个IP
  k8s:
    - 10.0.0.1
    - 10.1.96.48
    - 10.1.34.74
    - 10.1.34.75
    - 10.1.34.68
    - 10.1.34.69
    - 10.1.34.70
    - 10.1.34.71
    - 10.1.34.72
    - 10.1.34.73
  # 包含所有etcd节点IP
  etcd:
    - 10.1.34.69
    - 10.1.34.71
    - 10.1.34.73

四、测试部署情况

1、访问Dashboard

随便访问任何一个节点，https://Node:30001 ，并输入上面输出的token。

登录后，点击Nodes，可以看到加入到k8s集群的所有节点，默认命名空间为default。

进入页面后，点击左侧导航栏Nodes，可以看到目前加入k8s的服务器节点。

2、使用kubectl命令查看容器

# 在任意一个master节点使用命令查看nodes
[root@k8s-master1 ~]# kubectl get nodes
NAME           STATUS   ROLES    AGE    VERSION
k8s-master1    Ready    <none>   29h    v1.18.6
k8s-master2    Ready    <none>   29h    v1.18.6
k8s-master3    Ready    <none>   29h    v1.18.6
k8s-node1      Ready    <none>   29h    v1.18.6
k8s-node2      Ready    <none>   29h    v1.18.6
k8s-node3      Ready    <none>   29h    v1.18.6

至此，多Master的k8s集群已经搭建成功。

五、增加集群节点

1、修改hosts文件添加新节点ip

# vi hosts

[newnode]
10.1.34.20 node_name=PaasNexus01
10.1.34.7  node_name=PaasHarbor01

2、执行部署命令

ansible-playbook -i hosts add-node.yml -uroot -k

3、在任一Master节点允许颁发证书并加入集群
